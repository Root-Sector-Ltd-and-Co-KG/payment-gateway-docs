## Request Flow — Checkout Creation

```mermaid
sequenceDiagram
    participant M as Merchant Backend
    participant AB as Admin Backend
    participant DB as MongoDB
    participant MF as Main Frontend
    participant P as Payment Provider

    M->>AB: POST /v1/checkouts/{siteId}/create
    AB->>AB: Validate API key scope (checkout:create)
    AB->>DB: Create checkout session record
    AB-->>M: { checkoutUrl, checkoutId }

    Note over M: Redirect customer to checkoutUrl

    M->>MF: Customer opens checkout page
    MF->>MB: GET /api/v1/checkouts/{id}
    MB->>DB: Fetch session + branding
    MB-->>MF: Session data + provider config

    MF->>MF: Render payment form

    MF->>P: Customer submits card details
    P-->>MF: Payment token / redirect URL

    MF->>MB: POST /api/v1/transactions/create
    MB->>DB: Create transaction record (initiated)
    MB->>P: Create payment intent
    P-->>MB: Payment confirmation
    MB-->>MF: { transactionId, status, redirectUrl }
```

## Request Flow — Webhook to Transaction Update

```mermaid
sequenceDiagram
    participant P as Payment Provider
    participant MB as Main Backend
    participant DB as MongoDB
    participant W as Your Application

    P->>MB: POST /hooks/stripe/{providerId}
    MB->>MB: Verify HMAC signature
    MB->>DB: Fetch provider + decrypt webhook secret
    MB->>MB: Parse event type (payment_intent.succeeded)
    MB->>DB: Update transaction status to succeeded
    MB->>DB: Create invoice if configured
    MB->>W: POST IPN payload to Site webhook URL
    W-->>MB: 200 OK
    MB-->>P: 200 OK
```

## Encryption Key Hierarchy

```mermaid
graph TD
    ENV["SYSTEM_ENCRYPTION_KEY\nenvironment variable"]

    ENV -->|encrypts credentials for| SKMSCreds["KMS Provider Credentials\nstored encrypted in MongoDB"]
    SKMSCreds -->|used by| SKMS["System KMS Provider\nAWS / Azure / GCP / Vault"]
    SKMS -->|manages| SDEK["System DEK\nData Encryption Key"]

    SDEK -->|encrypts| SysData["System-Level Data\nUser accounts\nOrganization settings\nSystem configuration"]

    ENV -->|encrypts credentials for| OKMSCreds["Org KMS Credentials\nper-org, stored encrypted"]
    OKMSCreds -->|used by| OKMS["Organization KMS Provider\nper-org, optional BYOK"]
    OKMS -->|manages| ODEK["Organization DEK\nper-org"]

    ODEK -->|encrypts| OrgData["Organization-Level Data\nTransaction records\nCustomer information\nPayment details"]
```

## Transaction Lifecycle

```mermaid
stateDiagram-v2
    [*] --> initiated : Customer submits checkout
    initiated --> pending_provider : Async payment GoCardless or Wire
    initiated --> succeeded : Instant payment Stripe card
    initiated --> failed : Payment rejected
    pending_provider --> succeeded : Provider webhook confirms
    pending_provider --> failed : Provider webhook fails
    succeeded --> refunded : Refund processed
    refunded --> [*]
    succeeded --> [*]
    failed --> [*]
```
